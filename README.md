# 후보 A. 피싱시그널

## 1. 프로젝트 한 줄 소개
보이스피싱 의심 문자나 전화 상황에서 사용자가 위험 신호를 확인하고, 가족 공유와 대처 가이드를 받을 수 있는 안전 UX 서비스입니다.

## 2. 기획 배경
토스 멘토링 시작 전 금융 교육에서 전화 기반 금융 범죄 사례를 학습했습니다. 보이스피싱은 피해자가 위험 신호를 판단하기 전에 심리적으로 압박하는 방식으로 이루어지며, 특히 고령층이나 금융 앱 사용에 익숙하지 않은 사용자는 의심 문자, 전화, 링크를 즉시 판별하기 어렵습니다.

단순 탐지 앱이 아니라 가족 공유, 쉬운 모드, 대응 체크리스트 중심의 안전 UX로 차별화합니다.

## 3. 문제 정의
- 의심 문자나 전화를 받았을 때 위험 여부를 빠르게 판단하기 어렵습니다.
- 보이스피싱 유형과 대처법이 흩어져 있어 실제 상황에서 활용하기 어렵습니다.
- 고령층 사용자는 복잡한 금융/보안 용어를 이해하기 어렵습니다.
- 가족에게 상황을 공유하고 도움을 요청하는 과정이 번거롭습니다.

## 4. 타깃 사용자
- 보이스피싱 위험에 취약한 고령층
- 부모님의 금융 사기 피해를 걱정하는 보호자
- 의심 문자나 전화를 받은 일반 사용자

## 5. 핵심 기능

### 5-1. 의심 문자 위험도 체크
사용자가 문자 내용을 입력하면 위험 키워드를 탐지해 위험도를 보여줍니다.

예시 키워드: 검찰, 경찰, 계좌이체, 안전계좌, 본인인증, 해외결제, 앱 설치, 저금리 대출

### 5-2. 피싱 유형별 시나리오 카드
대표적인 보이스피싱 유형을 카드 형태로 제공합니다.

예시 유형: 검찰 사칭형, 가족 사칭형, 택배 문자형, 카드 해외결제형, 저금리 대출형, 지인 사칭 메신저형

### 5-3. 가족 공유 문구 생성
의심 상황을 가족에게 쉽게 공유할 수 있도록 메시지를 자동 생성하고 복사할 수 있게 합니다.

> 예시: "방금 의심스러운 문자를 받았어요. 내용은 해외결제 확인 링크이고, 위험도가 높게 나왔습니다. 링크를 누르지 않고 먼저 가족에게 확인하려고 합니다."

### 5-4. 대처 체크리스트
위험 상황에서 즉시 행동할 수 있는 체크리스트를 제공합니다.

- 통화를 끊습니다.
- 링크를 누르지 않습니다.
- 앱을 설치하지 않습니다.
- 가족이나 지인에게 먼저 확인합니다.
- 경찰 또는 금융기관에 신고합니다.

### 5-5. 쉬운 모드
고령층 사용자를 위해 큰 글씨, 쉬운 문장, 명확한 색상으로 위험 정보를 보여줍니다.

## 6. MVP 범위
- 문자 내용 입력 및 위험 키워드 탐지
- 위험도 결과 카드
- 피싱 유형별 카드 목록
- 가족 공유 문구 생성
- 대처 체크리스트
- 반응형 웹 배포

## 7. 기술 스택
| 영역 | 기술 |
|---|---|
| Frontend | React, TypeScript, Vite |
| Styling | Tailwind CSS |
| State Management | Zustand |
| Database | Supabase |
| Deploy | Vercel |
| Additional APIs | Clipboard API, Web Share API |

## 8. 6주 개발 계획
| 주차 | 내용 |
|---|---|
| 1주차 | 기획 확정, 와이어프레임, 프로젝트 세팅 |
| 2주차 | 홈 화면, 문자 입력 UI, 결과 카드 UI |
| 3주차 | 위험 키워드 탐지 로직, 위험도 계산 |
| 4주차 | 시나리오 카드, 대처 체크리스트 |
| 5주차 | 공유 기능, 기록 저장, 반응형 UI |
| 6주차 | QA, 리팩토링, 배포, README 정리 |

## 9. 기대 효과
- 사용자가 의심 상황에서 즉시 위험 신호를 확인할 수 있습니다.
- 고령층 사용자의 금융 사기 피해 예방에 도움을 줄 수 있습니다.
- 가족 간 안전 확인과 공유 과정을 쉽게 만들 수 있습니다.

## 10. 차별화 포인트
단순 스팸 차단 앱이 아니라 위험 신호 감지 → 가족 공유 → 대처 행동까지 이어지는 UX 흐름에 집중합니다. 쉬운 모드를 통해 디지털 취약 계층도 직관적으로 사용할 수 있도록 설계합니다.

# 후보 B. 오늘의 루트

## 1. 프로젝트 한 줄 소개
요구르트 아주머니처럼 정기 방문을 하는 이동 노동자가 고객 위치, 방문 시간대, 상품 특성, 우선순위를 기반으로 하루 방문 순서를 효율적으로 관리할 수 있도록 돕는 지도 기반 루트 관리 서비스입니다.

## 2. 기획 배경
요구르트, 우유, 반찬, 도시락처럼 정기 방문 판매를 하는 이동 노동자는 고객 위치, 방문 가능 시간, 상품 신선도, 부재 여부를 함께 고려해야 합니다. 하지만 이러한 정보는 경험이나 수기 메모에 의존하는 경우가 많아 이동 동선이 비효율적이거나 재방문 부담이 커질 수 있습니다.

이 서비스는 방문형 이동 노동자의 하루 업무를 지도와 리스트로 시각화하고, 다음 방문 우선순위를 추천해 불필요한 이동과 업무 부담을 줄이는 것을 목표로 합니다.

## 3. 문제 정의
- 고객별 방문 가능 시간과 요청사항을 한눈에 보기 어렵습니다.
- 방문 순서가 비효율적이면 이동 시간이 증가합니다.
- 냉장 상품은 오래 들고 다닐수록 품질 관리 부담이 커집니다.
- 부재 고객이나 재방문 고객을 따로 관리하기 번거롭습니다.
- 하루 업무 진행률과 남은 업무량을 파악하기 어렵습니다.

## 4. 타깃 사용자
- 요구르트, 우유, 반찬 등 정기 방문 판매자
- 방문 배송을 직접 관리하는 소상공인
- 정해진 구역을 반복적으로 방문하는 이동 노동자

## 5. 핵심 기능

### 5-1. 오늘의 방문 고객 리스트
오늘 방문해야 할 고객을 카드 형태로 보여주고, 방문 예정/완료/부재/재방문 상태를 관리합니다.

고객 카드 정보: 고객명, 방문 가능 시간대, 주문 상품, 요청사항, 현재 상태

### 5-2. 지도 기반 고객 위치 확인
고객 위치를 지도에 마커로 표시하고, 상태별로 마커 색상을 다르게 보여줍니다.

상태 구분: 방문 예정 / 우선 방문 / 방문 완료 / 부재 / 재방문 필요

### 5-3. 우선 방문 추천
현재 위치, 방문 가능 시간, 냉장 상품 여부, 고객 우선순위를 기준으로 다음 방문 고객을 추천합니다.

추천 기준: 현재 위치와의 거리, 방문 마감 시간까지 남은 시간, 냉장 상품 여부, 고객 중요도, 재방문 여부

### 5-4. 방문 상태 관리
방문 후 상태를 버튼 하나로 빠르게 업데이트할 수 있습니다.

액션: 방문 완료 / 부재 / 재방문 / 메모 추가

### 5-5. 하루 업무 요약 대시보드
방문 완료율, 남은 고객 수, 부재 고객 수, 예상 남은 시간, 이동 거리 등을 요약해서 보여줍니다.

## 6. MVP 범위
- 고객 mock 데이터 구성
- 지도 위 고객 마커 표시
- 오늘의 방문 리스트
- 방문 상태 변경
- 우선순위 점수 기반 추천
- 업무 요약 카드
- 모바일 반응형 웹 배포

## 7. 기술 스택
| 영역 | 기술 |
|---|---|
| Frontend | React, TypeScript, Vite |
| Styling | Tailwind CSS |
| Map | Kakao Maps API |
| State Management | Zustand |
| Server State | TanStack Query |
| Database | Supabase |
| Chart | Recharts |
| Deploy | Vercel |

## 8. 6주 개발 계획
| 주차 | 내용 |
|---|---|
| 1주차 | 기획 확정, mock 데이터 설계, 와이어프레임, 프로젝트 세팅 |
| 2주차 | 지도 API 연동, 고객 마커 표시, 고객 카드 UI |
| 3주차 | 방문 리스트, 상태 변경, 필터링 구현 |
| 4주차 | 우선순위 점수 계산, 추천 방문 카드 구현 |
| 5주차 | 업무 요약 대시보드, 차트, 반응형 UI |
| 6주차 | QA, 리팩토링, 배포, README 정리 |

## 9. 기대 효과
- 이동 노동자의 불필요한 이동 시간을 줄일 수 있습니다.
- 고객 부재나 재방문 상황을 체계적으로 관리할 수 있습니다.
- 정기 방문 업무를 데이터 기반으로 시각화할 수 있습니다.
- 소규모 방문 판매자의 디지털 업무 전환을 도울 수 있습니다.

## 10. 차별화 포인트
단순 길찾기가 아니라 방문 가능 시간, 상품 특성, 고객 우선순위, 재방문 상태를 함께 고려하는 업무 관리형 루트 서비스입니다. 복잡한 최단 경로 알고리즘 대신 MVP에서는 점수 기반 우선순위 추천으로 범위를 현실적으로 조절합니다.

# 후보 C. 빵나왔어요

## 1. 프로젝트 한 줄 소개
동네 빵집의 갓 나온 빵 정보를 지도와 타임라인으로 확인하고, 좋아하는 빵이 나올 때 알림을 받을 수 있는 로컬 상권 서비스입니다.

## 2. 기획 배경
동네 빵집은 프랜차이즈보다 개성 있는 메뉴와 갓 구운 빵의 매력이 있지만, 소비자는 원하는 빵이 언제 나오는지 알기 어렵습니다. 인기 메뉴는 방문했을 때 이미 품절되어 있을 수 있고, 빵집마다 정보 채널이 인스타그램, 네이버 플레이스, 전화 문의 등으로 흩어져 있습니다.

이 서비스는 소비자가 갓 나온 빵을 놓치지 않도록 돕고, 동네 빵집에게는 제품을 빠르게 알릴 수 있는 가벼운 홍보 채널을 제공합니다.

## 3. 문제 정의
- 내가 좋아하는 빵이 언제 나오는지 알기 어렵습니다.
- 인기 메뉴는 방문했을 때 이미 품절될 수 있습니다.
- 동네 빵집 정보가 여러 채널에 흩어져 있습니다.
- 빵집 투어를 하려면 동선과 시간을 직접 계산해야 합니다.
- 소규모 빵집은 디지털 홍보 수단이 부족합니다.

## 4. 타깃 사용자
- 갓 나온 빵을 사고 싶은 동네 소비자
- 특정 빵이 나오는 시간을 알고 싶은 사용자
- 주말에 빵집 투어를 하고 싶은 대학생/직장인
- 동네 빵집을 홍보하고 싶은 소상공인

## 5. 핵심 기능

### 5-1. 지도 기반 동네 빵집 탐색
사용자 주변 또는 선택한 지역의 빵집을 지도 위에 표시합니다.

기능: 빵집 위치 마커 표시, 빵집 이름·대표 메뉴·영업 여부 확인, 마커 클릭 시 빵집 카드 노출, 오늘 나온 빵 요약 표시

### 5-2. 실시간 빵 타임라인
각 빵집에서 빵이 나오는 시간을 타임라인 형태로 보여줍니다.

상태 구분: 곧 나와요 / 방금 나왔어요 / 지금 살 수 있어요 / 품절 가능성 있어요

### 5-3. 최애 빵 알림 예약
사용자가 좋아하는 빵을 등록하면 해당 빵이 나오는 시간에 알림을 받을 수 있습니다.

MVP에서는 실제 푸시 알림 대신 앱 내부 알림 목록과 브라우저 Notification API로 구현합니다.

### 5-4. 빵집 투어 동선 만들기
사용자가 가고 싶은 빵집을 선택하면 투어 코스를 만들고 예상 이동 시간을 보여줍니다.

기능: 빵집 여러 개 선택, 거리와 예상 이동 시간 표시, 지도 위 경로 선 표시

### 5-5. 빵 카드 애니메이션
갓 나온 빵은 따뜻한 색상과 연기 애니메이션으로 표현합니다.

| 경과 시간 | 상태 | 표현 |
|---|---|---|
| 0~30분 | 갓 나옴 | 연기 애니메이션, 주황색 강조 |
| 30분~2시간 | 판매 중 | 기본 카드 |
| 2시간 이후 | 식었을 수 있음 | 회색 톤 |
| 품절 | 품절 | 카드 비활성화 |

## 6. MVP 범위
- 지도에서 빵집 보기
- 빵집별 빵 타임라인 보기
- 현재 시간 기준 빵 상태 표시
- 최애 빵 알림 등록
- 투어에 빵집 추가
- 반응형 UI 및 배포

- 소비자가 갓 나온 빵을 놓치지 않고 구매할 수 있습니다.

- 지역 소상공인과 소비자를 연결해 로컬 상권 활성화에 기여할 수 있습니다.
